## 📋 COMPLETE BUILD CHECKLIST STRUCTURE

### Phase 1: Master Data Tables (Build First)
1. **Locations** - 4 Southwest offices (MUST BE FIRST)
   - Why first: Projects table requires Location_Code lookup
   - Pre-populated data: SD, LAS, PHX, DEN
   - Location_Code is TEXT primary key (not auto-number)

2. **Apparatus_Type_Master** - 132 apparatus types
   - Updated structure: 4 NETA columns (ATS/MTS sections and hours)
   - Verification steps for existing table
   - Migration guidance if needed

### Phase 2: Operational Tables (Field Tech Access)
3. **Projects** - Core project records
   - Requires Locations lookup
   - Job_Number is primary business key

4. **Scopes** - Work breakdown
   - **NETA_Standard field (ATS/MTS)** ⭐
   - Requires Projects lookup
   - Determines testing specifications for all apparatus

5. **Tasks** - Work organization
   - **IMMEDIATE IMPLEMENTATION** ⭐
   - Manual creation by PMs (NOT imported)
   - Requires Scopes and Projects lookups

6. **Apparatus** - Individual testable units
   - **ZERO FINANCIAL FIELDS** ⭐
   - Operational data only
   - Requires Projects, Scopes, Tasks, Apparatus_Type_Master lookups

### Phase 3: Financial Tables (Management Only - RESTRICTED)
7. **Scope_Financial_Configuration** - Rates and multipliers
   - 1:1 with Scopes
   - Contains sensitive pricing data
   - **Field techs have NO ACCESS**

8. **Apparatus_Revenue** - Revenue recognition
   - 1:1 with Apparatus
   - Auto-generated by Power Automate
   - **Field techs have NO ACCESS**
