## Decision-Ready Closeout Shape

Desktop Codex closeouts should use this compressed shape:

1. `Status`: exactly one queue status.
2. `Decision needed`: one sentence naming what Jason or VS Code Codex must decide.
3. `Recommendation`: proceed, park, revise, or abort.
4. `What was read`: exact files and any source-domain roots inspected.
5. `What was written`: exact files written.
6. `What remains forbidden`: short authority confirmation.
7. `Evidence`: validation command and result.
8. `Graduation impact`: whether this lane should unlock another lane.

This gives Jason an outcome to approve instead of a transcript to reconstruct.
